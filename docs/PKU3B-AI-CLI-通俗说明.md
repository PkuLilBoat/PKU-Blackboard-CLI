# PKU3B AI CLI 通俗说明

这份文档用大白话说明：这个项目现在做到哪了、能拿来干什么、为什么这次改造是有意义的。

## 1. 这个项目现在是什么

简单说，它还是那个 `pku3b`，但已经开始往“更适合 AI 和脚本调用”的方向升级。

以前它更像一个给人手动在终端里用的小工具。

现在它正在变成两用版本：

- 人可以直接在终端里用；
- AI、脚本、自动化工具也可以稳定调用。

这次改造没有推翻重写，而是在原来的 Rust CLI 基础上继续加能力。

## 2. 这次改造到底改了什么

最重要的不是“多了几个命令”，而是命令的输出方式变了。

以前很多命令主要是给人看，输出更偏终端展示。

现在一批高价值命令已经支持稳定 Markdown 输出，这意味着：

- AI 更容易读懂结果；
- shell 脚本更容易继续处理；
- 以后做 GUI、TUI、网页壳子也更方便复用。

已经完成 Markdown 化的重点命令包括：

- `course`
- `cache`
- `announcement`
- `document`
- `coursetable`
- `assignment`
- `find`
- `search`
- `tree`
- `video`

## 3. 你可以把它理解成什么

可以把现在的 `pku3b` 理解成一个“北大教学网低层接口工具箱”。

它不负责替你判断“什么最重要”，也不负责替你总结。

它更像是：

- 帮你列出资源；
- 帮你按 ID 取详情；
- 帮你查找标题；
- 帮你下载文件或视频；
- 把结果稳定地吐给人或者 AI。

也就是说，它做“取数据”和“做原子操作”，不做“高层智能决策”。

## 4. 这次为什么不是重写

因为原来的 Rust 版本本身已经有很多真实可用的底层能力：

- 登录流程
- 作业下载与提交
- 视频下载
- 课表
- 选课相关能力
- 缓存和配置

所以正确路线不是另起炉灶，而是：

- 保留 Rust 版现有可用能力；
- 借鉴 `pku3b_AI` 里更结构化的资源设计；
- 把这些能力搬进同一个单文件 CLI 里。

这样做的好处是：

- 不需要 Python 运行时；
- 不需要 MCP；
- 不需要常驻服务；
- 最终交付仍然是一个本地单二进制工具。

## 5. 现在已经能稳定拿来做什么

如果你是普通用户，现在它仍然可以继续当命令行工具直接用。

如果你是 AI 或自动化调用方，现在可以更放心地拿这些命令做后端：

- 列课程
- 列公告
- 看公告详情
- 列文档
- 看文档详情
- 查标题
- 查树节点
- 列作业
- 下载作业附件
- 列视频
- 下载视频
- 拉课表

## 6. 登录这块现在怎么理解

这次还把登录策略理顺了。

按真实使用习惯，应该优先模拟教学网里的“校园卡用户”入口：

- 登录页：`https://course.pku.edu.cn/webapps/login/`
- 优先走校园卡用户路径
- 先尝试直接登录
- 只有当上游真的返回 OTP 拒绝时，才要求补 OTP

对你这个账号来说，Blackboard 常规流程通常不需要 OTP，这个行为已经写进说明和测试文档了。

## 7. 目前完成度怎么判断

如果用通俗的话说：

- 主方向已经立住了；
- 关键命令面已经打通了；
- Markdown 合同已经不是口头承诺，而是有代码、有测试、有真人验证；
- 这一个里程碑可以算完成。

但这不代表“永远不用再动”。

更准确地说，是：

- 第一阶段已经完成；
- 后面还可以继续扩命令面、补更深的集成测试、优化文档和交互。

## 8. 这次已经实际验证了什么

不是只改文档，也不是只跑单测。

这次已经实际做过：

- Rust 单元测试
- 真账号登录测试
- Markdown 输出验证
- 文档/公告/作业/视频/课表等关键命令 smoke test
- 视频真实下载产物验证

所以现在这个“已完成当前里程碑”的说法，是有证据支撑的。

## 9. 如果你只想看最关键的一句话

一句话版本：

`pku3b` 现在已经从“一个给人手动用的北大教学网命令行工具”，明显推进到了“一个既能给人用、也能给 AI 和脚本稳定调用的单文件 Rust CLI 后端”。
